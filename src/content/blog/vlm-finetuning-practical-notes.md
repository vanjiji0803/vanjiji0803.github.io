VLM，也就是 vision-language model，微调起来比纯文本大模型更容易给人一种“已经对齐了”的错觉。模型能看图，能回答，输出也像那么回事。但真正部署到业务场景时，经常会遇到几个熟悉的问题：图里关键区域没看见，文本描述很流畅但证据不对，目标位置说错，OCR 漏字，遇到分辨率变化就退化，或者在多轮对话里逐渐忘记最初的视觉约束。

所以 VLM 微调不能只看最终回答是否顺眼。它要同时回答三个问题：模型看到了什么，模型如何把视觉证据映射到语言空间，模型是否按照任务约束输出。

一个典型 VLM 架构可以粗略拆成三层。第一层是视觉编码器，比如 ViT、CLIP vision encoder 或者 SigLIP，把图像切成 patch 并编码成视觉 token。第二层是投影层或 connector，把视觉特征映射到语言模型可消费的 hidden space。第三层是 LLM decoder，负责根据视觉 token 和文本指令生成答案。微调时到底训练哪一层，决定了成本、效果和风险。

如果任务只是让模型适应某种问答格式，通常 LoRA 放在语言模型部分就够了。比如医疗器械质检图像里，输出必须是固定 JSON：缺陷类型、位置、置信判断、是否需要人工复核。这类任务的核心是输出习惯和判断边界，未必需要大幅更新视觉编码器。

如果任务依赖细粒度视觉差异，比如细小裂纹、烟雾边缘、手术器械尖端、病理切片里的微结构，仅调 LLM 可能不够。问题可能出在视觉表示本身，也可能出在 connector 没有把局部视觉信号稳定传给语言模型。这时需要考虑更高分辨率输入、区域裁剪、视觉编码器部分解冻、或者加入定位辅助任务，而不是一味增加问答样本。

数据构造是 VLM 微调最关键的环节。纯文本 SFT 里，输入输出错一点可能还能靠语言先验补上；VLM 里，一条错误标注会同时污染视觉 grounding 和语言行为。高质量数据至少要包含四类样本。

第一是直接问答：图中有什么、状态是什么、是否异常。它训练基础识别能力，但很容易让模型学成“看图说话”。

第二是定位和证据：要求模型指出依据来自图像中的哪个区域，或者描述可见证据。即使最终系统不输出 bounding box，训练时加入位置或区域描述，也能减少纯语言幻觉。

第三是反事实和难负例：图中没有某个目标、目标被遮挡、画面模糊、相似类别混淆。没有这类数据，模型会倾向于迎合问题，用户问什么它就找什么。

第四是结构化输出：比如 JSON、表格、标签集合或分类字段。生产系统通常不需要一段优美散文，而需要可解析、可回放、可统计的结果。

微调策略上，LoRA/QLoRA 仍然是最现实的起点。QLoRA 能降低显存压力，让 7B 或 13B 级别的 VLM 在单机多卡甚至较小资源上做实验。但需要注意，VLM 的显存不只来自语言模型，图像分辨率、patch 数量、batch size、序列长度都会推高成本。很多时候，显存瓶颈不是参数量，而是视觉 token 加长后 attention 计算爆掉。

训练时我会先固定视觉编码器，只调 connector 和 LLM 的 LoRA，建立一个 baseline。这个 baseline 的意义不是追求最好，而是判断问题到底在哪里。如果 baseline 在格式、术语、任务流程上明显改善，但细粒度视觉判断仍然差，说明视觉侧可能需要改；如果视觉判断还行但输出不稳定，问题更多在语言侧和数据格式。

评测不能只用 BLEU、ROUGE 或者人工看几条样例。VLM 至少应该拆成几组指标：分类准确率、定位误差、OCR 字段准确率、结构化输出合法率、拒答正确率、跨分辨率鲁棒性、遮挡和低光照下的退化曲线。对于 agent 场景，还要看下游工具调用是否正确。比如模型识别到某个异常后，是否生成了正确的 tool 参数，而不是只在自然语言里说“可能异常”。

一个常见失败模式是语言模型过强，视觉证据过弱。它会根据问题和常识补全答案，看起来很自然，但不一定来自图像。解决这个问题不能只靠 prompt 里写“请基于图片回答”。更有效的方法是加入难负例、要求证据描述、训练拒答样本，并在评测中加入“图中不存在但问题暗示存在”的 case。

另一个失败模式是过拟合标注风格。比如训练集中所有异常都写成“建议复查”，模型上线后无论轻重都给同一句话。这种问题通常不是模型能力不够，而是数据分布太单一。解决方法是把标签体系拆细，加入严重程度、可见性、置信来源和处理建议，让模型学习更清楚的决策边界。

我对 VLM 微调的一个基本判断是：不要把它当成“给图片加一个聊天入口”。真正有价值的 VLM 系统，应该把视觉理解、语言推理和工程输出连起来。图像提供证据，语言模型负责解释和组织，结构化 schema 负责让结果进入后续系统。

如果一句话总结：VLM 微调不是让模型更会描述图片，而是让它在特定任务里更稳定地使用视觉证据。只评估文字答案会高估模型，只评估分类准确率又会低估系统风险。好的 VLM 微调，需要把数据、架构、训练范围和评测闭环一起设计。
